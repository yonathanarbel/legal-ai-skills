# Security Policy

Please report suspected credential exposure or a vulnerability privately through GitHub's security-advisory feature rather than a public issue.

Do not submit client documents, confidential research queries, authentication tokens, private database contents, or internal infrastructure details in issues or pull requests. Examples and tests must use synthetic, public-domain, or appropriately licensed material.

The `legal-bibliography` skill can send query text to a configured remote service. Review that service's privacy and retention policies before using confidential facts. The other bundled command-line tools operate on local files unless the user separately configures remote storage or execution.
